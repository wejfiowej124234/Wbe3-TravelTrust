**定稿检查勾选**（全部满足后方可视为 SSOT 闭合）：□ 上表每行 value 已填实（无数值/枚举的须写 Runbook §N 或表名等可落点）□ 每行 evidence_pointer 已填 □ who_can_change 无「—」□ 08-4 对应承诺句与本表 key 一致 □ 变更记录表已启用且关键 key 变更已登记。

**SSOT 与链/DB 冲突时的优先级**（写死）：**链事件 > SSOT（08-3）> DB**。若 SSOT 与链状态冲突，**以链为准**；若 DB 与链/SSOT 冲突，以链与 SSOT 为准并修正 DB。运行冲突时不得现场裁量，须按此优先级执行；**冲突时自动停机策略**（链参数与 08-3 文档冲突且无法自动修正时拒绝执行或暂停相关结算）见 Runbook §11。写死于 Runbook §11。

**权限层级（紧急 vs 常规）**（无「终极钥匙」、可审计）：
- **常规多签**：改 08-3 参数须 Timelock + 双人审批；受 paramFreezeDays、paramChangeMaxPer30d 约束。
- **紧急多签**：仅限 Runbook §7 列明的场景（Pause、OFAC/监管执行、安全事件）；可**豁免** paramFreezeDays 与 pauseCooldown，**不可**改 ofacHitPolicy 等法务口径 key；每次紧急动作须事后留痕（evidence 或 08-3 变更记录）。**不存在**「超级多签」或单把「终极钥匙」— 紧急多签仍为多签门限，且权限范围写死于 Runbook。
- **paramFreezeDays 防绕开 / 全局变更指数**（写死）：paramChangeMaxPer30d 按**关键 key 变更次数**计；**不得**以「不同 key 轮流改」或**不同参数交替改动**绕过（如改 disputeWindow、改 stakeTier、改 arbFee 各一次—单项未超限但整体已重构系统，合计仍计次数）。**全局变更指数（可执行口径）**：**30 天内**以「08-3 变更记录表」中**日期**落在同一滚动 30 日窗口内的**记录条数**计；每条变更记录计 **1**（同一批次变更多 key 仍计 1 条）；若某条记录涉及多个 param_key，**仍计 1 条**，不得拆成多条规避。**指数 ≤ paramChangeMaxPer30d（3）** 方可执行下一次非紧急参数变更；超限则触发 paramFreezeDays 冻结期。执行处：变更前由产品/风控在 Runbook §10 或本表变更记录中核对「过去 30 日记录条数」；无此口径则频率限制可被技术性规避，企业级必问。**参数变更触发冻结/暂停治理**（写死，可加强）：参数修改触发冻结期（paramFreezeDays 已有）；**连续修改触发暂停治理权**（如超 paramChangeMaxPer30d 后 N 天内禁止非紧急参数变更）由风控/产品定稿写死 Runbook §10 或本节，防参数滥用。**导游 unstake 冷却期**（若存在）：若设计**导游退出质押冷却期**（如 7～14 天）以减缓**导游质押挤兑**瞬间倾泻，须写死天数并落 Runbook；由产品/风控定稿。

**证据删除与司法协助**（写死）：evidenceRetentionDays 届满后可删原文件；**hash + receipt 永久保留**（可证明「曾存在、何时、何人」）。删除后司法协助：平台可向司法机构提供 hash、receipt、上传者与时间等元数据；不保留可逆原文件。外部备份策略（若有）见 Runbook §11。

**证据存储物理级冗余**（写死，企业级）：① **多地域/跨云**：证据存储须多地域或跨云备份（具体见 Runbook §11）；单点故障不得导致证据不可用。② **S3/存储故障达 7 天**：若主存储不可用超过 7 天，**暂停新仲裁受理**；存量争议**默认按「证据缺失」适用默认裁决**（退款优先等）；恢复后按既有规则继续。③ cold archive 策略（若有）见 Runbook §11。无物理冗余要求 = 仲裁存在物理单点。

**脱锚时争议金额计价**（写死）：稳定币大幅脱锚时，争议金额按**裁决时链上价格**计算；与 08-4 第 5 章一致。

**reorg 超过 finalityN（超 finality 场景）**（写死）：若链重组深度 **> finalityN**（如 > 12 block）：**自动暂停结算**；须**人工确认 finality** 后恢复；**已基于旧链状态执行的裁决/DB 须按 Runbook §11 执行回滚或重放**（不得保留与最终链不一致的终态）。流程见 Runbook §11。未写死则 reorg 攻击可穿透。

**服务开始时间防篡改**（写死）：serviceStartTimeSource=backend_evidence 时，**backend 时间源须可验证**（如服务器时间签名、NTP 可信源或时间戳公证）；否则仲裁时间可被攻击。具体实现由运维/架构定稿并留 08-3 变更记录或 Runbook §11。

**质押与规模动态**（写死）：当前 **stakeTierThresholds / stakeToOrderCapMap 为静态档位**；**GMV 或规模上升 10×/50× 时须触发重估**（是否自动提高质押比例由治理/路线图决定，当前无自动调整函数）；若上线后规模扩大须评估 W-DEFAULT-STAKING 压力测试与动态调整，否则存在规模暴露风险。**质押覆盖率公式**（写死，企业级必答）：须定义 **覆盖率 = 质押总额 / 在途订单风险暴露**（或等效公式），并写死 **≥ X%** 的阈值；无公式则无法证明不会穿透质押池，审计必问。**默认裁决比例失衡是否会击穿质押池**（写死）：若默认裁决长期偏向游客或偏向导游，**是否击穿质押结构**须有**覆盖率公式、风险暴露曲线、最大坏账模型**写死；否则为潜在资损模型缺口。具体公式与 X 由风控/产品定稿写死 Runbook 定稿表或本节。**默认裁决击穿模拟（4️⃣）**（写死）：假设**80% 争议证据不足、默认裁决规则为退款优先**—**导游经济模型是否崩溃**须做**默认裁决击穿模拟**；**质押覆盖率压力测试**须写死（含灰色证据比例升高场景）。若未做，系统在灰色证据比例高时会经济性坍塌；由风控定稿写死 Runbook 定稿表或本节。**默认裁决连续偏向**（写死）：若**连续 N 单全部退款或全部放款**（如 100 单），是否击穿质押池、导游退出、游客薅羊毛、资金池枯竭须有**连续偏差阈值**触发**自动风控冻结**（如暂停该类别新单、强制复核、或提高质押/限单）；由风控定稿写死 Runbook §11 或本节，否则极端连续偏向无制衡。**质押覆盖率 worst-case 模型（4️⃣）**（写死）：须有**最坏情形模型**—**5 个导游同时违约、单导游最大并发、单导游最大金额、同地区集中违约**下，**质押覆盖率是否可能 < 最大赔付**；若存在击穿概率须写死：**上限**（如单笔/单导游赔付上限）、**动态风控**（如超阈值自动提高质押要求）、**自动暂停**（如覆盖率低于 X% 暂停新单）。无 worst-case 模型=经济安全未闭合；由风控定稿写死 Runbook 定稿表或本节。**争议量 10× 触发阈值（7️⃣）**（写死）：当争议量达 10×（如订单 10,000→争议 800 级）时，**SLA 自动延长/自动暂停新争议/动态提高 arbFee** 的**触发阈值**（如待裁件数 > N 或争议率 > X%）须写死；由风控/运营定稿 Runbook §11 或本节，否则属规模错觉。

**仲裁费博弈防护**（写死）：**低成本骚扰**：arbFeeBase/arbFeeCap 与 arbFeeMultiplier 已限制单次成本；**高额压制小额**：小额争议可设最低可裁门槛或费率曲线（见 Runbook §11 或 03 业务流程）。仲裁费曲线**当前不随争议率自动变化**；若发现博弈攻击可经治理调整参数。**仲裁费用途**（防逆向激励）：仲裁费**不得进入平台营收或与争议数量挂钩**；须仅用于运营/合规成本或进入公共池/治理金库（具体写死 08-4 第 3 章与治理文档）。博弈分析见风控文档。

**监管 KYC 全量升级路径**（写死）：若监管要求**全量实名/KYC**：① **仅改参数即可**时（如将 kycAmountThresholds 设为 0 或全量触发），走 08-3 变更流程 + 08-4 同步；② **需改架构**时（如新增 KYC 管线、数据留存范围），须走产品/法务评估并更新 08-3、08-4、Runbook。企业级须有「监管升级路径」文档或 Runbook §11。

**时间双轨冲突**（写死）：disputeWindow 以**链时间（block）**为界，裁决所采服务开始时间以 **serviceStartTimeSource（backend_evidence）** 为准；冲突时**窗口开闭以链为准、裁决采信以 08-3 为准**，见 08-4 第 3 章与 Runbook §11。

**代币标准/合约升级风险**（写死）：若**结算代币（如 USDC）升级合约或黑名单逻辑变更**，须**重新评估过门**（Gate-2/Gate-4 等）；是否触发重新过门由风控/法务在 Runbook 或 08-2 中写死。未写死则存在链上升级穿透风险。

**异常争议率与关联账户防护**（写死）：**异常争议率**（如某类服务/某地区争议率显著偏离均值）须触发告警或**自动冻结**该类别新单/复核（阈值与动作见 Runbook §11 或 03 风控）；**关联账户检测**（如多账号协作制造争议套利）须有检测与处置流程，否则存在「沉默攻击」风险。企业级须在 Runbook 或 03 中写死。

**证据存储与数据主权**（写死）：evidenceRetentionDays 等见上表；若证据**跨境传输**或涉及 **EU 用户**，须满足首发司法域与业务地数据合规（如 EU GDPR 传输限制、本地化）；跨境路径与 EU 合规口径须在隐私政策或 Runbook 中写死，见 08-4 第 3 章。
