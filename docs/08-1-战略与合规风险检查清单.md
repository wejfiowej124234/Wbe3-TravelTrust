## ⛔ 系统级不变量（宪法级条款）— 绝对不允许被破坏

在 17 条、12 缝、Gate、11 项、6 项、五类演化风险之上，以下 **5 条**为**系统宪法级条款**：任意违反即视为协议边界被突破，不得以「临时例外」silently 绕过。发版前必须逐条可证明满足。

| # | 不变量 | 含义 | 可验证方式 |
|---|--------|------|------------|
| **1** | **用户资金最终状态只能来自链事件** | 任意资金终态（已支付/已退款/已扣罚等）必须由链上事件驱动；DB/后端仅缓存与索引，不得单方面改写资金终态 | 01 链为准、17 条 #2/#12、对账设计、审计 |
| **2** | **任意人工操作必须可追溯** | 多签/仲裁/执行器/参数变更等人工操作须留痕（谁、何时、何内容、审批链）；不可删除、不可篡改 | 01 §7、17 条 #11、Runbook、08-3 变更记录 |
| **3** | **任意参数修改必须留痕** | 08-3 中参数变更须有变更记录（旧值→新值、理由、批准、生效时间）；并按规定触发 08-4 披露与 evidence | 08-3 变更记录、W-PARAM-ABUSE、W-PDP-SSOT-CONSISTENCY |
| **4** | **任意裁决必须可回放** | 裁决结论、依据证据、适用规则、签名与审批链可被事后审计与回放；不得「口头裁决无留痕」 | 01 §7、D2、Q 声誉、证据链三条硬规则 |
| **5** | **任意发布必须可验证** | 前端/合约/关键配置发布须有可验证产物（manifest、hash、可复现构建）；不得不可验证发布 | Q6、07、08-2 W-Q6-FE、W-BOMB-10 |

**违反后果**：上述任一条被突破且未走**显式例外流程（留痕+复盘+补救）**，视为门禁失效；发版前检查须包含「5 条宪法级不变量」逐条勾选。

## 如何把「检查」变成「闭合」（四列 + Gate）

**原则**：任何一条没有**四列**就算没闭合。真正卡上线的往往不是技术条目，而是 **Owner 不唯一 + Evidence 不存在**。

### 闭合四列（每条 ⬜/🟡 须填满）

| 列 | 要求 | 说明 |
|----|------|------|
| **Decision（写死结论）** | 二选一/三选一，**不可歧义**的一句话 | 例如：「仲裁费命中后执行【限额】」而非「视情况处理」 |
| **SSOT 落点** | 参数表 / 对外口径包 / Runbook **至少一个** | 结论必须落在可交付物上 |
| **Owner** | 法务 / 产品 / 风控 / 运维 / 架构 **只允许一个主责** | 谁负责定稿与签字 |
| **Evidence** | 链接到证据产物 | 演练记录、签字、配置快照、版本号、manifest hash 等 |

### 按 Gate 推进（主责与配合）

| Gate | 名称 | 08 节范围 | 主责 | 配合 |
|------|------|-----------|------|------|
| **Gate-1** | 合规 | A1、A2、A3、A4、A5、A6 | 法务 | 产品 |
| **Gate-2** | 密钥与资损 | B1、B2 | 运维 | 安全、架构 |
| **Gate-3** | 仲裁与声誉 | D2、D3 + Q 声誉三件套 | 运营（法务、产品配合） | — |
| **Gate-4** | 黑天鹅 | O、E1 + 稳定币/脱锚/Gas | 风控 | 运维 |
| **Gate-5** | 可替代性 | Q6 + 供应链/分发 | 架构 | 运维 |

### ⚠️ 定稿载体一致性（08-3 SSOT 与 08-4 对外口径）— 灾难级不一致防护

**问题**：08-3 = 技术参数真相源，08-4 = 对外公开口径。若 SSOT 写「ofacHitPolicy = 只读冻结」而 08-4 第 8 章写「允许退出但禁新增」，即**技术写 A、公关说 B**，属灾难级不一致。

**必须写死**：

| 机制 | 要求 | 落点 |
|------|------|------|
| **关键 key 与 08-4 章节映射** | 08-3 中与对外承诺强相关的 key（如 ofacHitPolicy、pauseCooldown、证据保留、仲裁费上限等）须**明确对应 08-4 章节**；变更 08-3 即须同步 08-4 对应处 | 08-3 表头或附录、08-4 版本号 |
| **CI 校验** | 若 08-3 中**关键 key** 被修改，PR 必须同时更新 08-4 对应章节或版本号并留痕；**否则阻断合并**（或显式豁免并留痕） | 仓库 CI、[08-2](08-2-附录-闭合工单表.md) W-PDP-SSOT-CONSISTENCY |
| **工单** | [08-2](08-2-附录-闭合工单表.md) **W-PDP-SSOT-CONSISTENCY**：关键 key 清单 + CI 规则 + 08-4 同步要求写死 | 08-2、CI 配置 |

### ⚠️ Gate 跨 Gate 冲突检测（单 Gate 闭合 ≠ 全 Gate 一致）

**问题**：Gate-1～5 单门闭合后，**Gate 之间可能逻辑冲突**。例如 Gate-4 黑天鹅允许迁移 token，Gate-1 合规规定不允许资产迁移——两门都闭合，但结论冲突。

**必须写死**：

| 机制 | 要求 | 落点 |
|------|------|------|
| **上线前全 Gate 横向一致性评审** | 发版前**至少一次**：全 Gate 横向一致性评审（Gate-1～5 结论互不冲突）；产出**会议纪要 + 冲突项结论表 + 证据包** | 07 发版前检查、[08-2](08-2-附录-闭合工单表.md) W-GATE-CROSS-CHECK |
| **工单** | [08-2](08-2-附录-闭合工单表.md) **W-GATE-CROSS-CHECK**：上线前必须有一次全 Gate 横向一致性评审会议纪要 + 证据包；未通过不得上线 | 08-2、发版 Checklist |
